  1. de grammatica zelf schrijven
  2. .g4 file compileren met ANTLR tool, normaal gealiast als antlr4. Dit produceert source code, inclusief .tokens files.
  3. in geval van een gecompileerde doeltaal: de source code compileren, bv. javac runnen
  4. optioneel: het resultaat testen. Dit gebeurt met een tool in de .jar, namelijk TestRig. Normaal is deze gealiast als grun. Inputs:
    1. naam van de grammatica
    2. naam van de regel waarmee we willen beginnen parsen (root van de syntax tree)
    3. eventuele opties, waaronder belangrijke:
      • tokens
      • tree
      • gui
      • encoding (van de input)
    Er zijn er ook handige voor het tracen van de constructie!
